Pre-Workshop Setup

Claude Code 101

Complete these steps before the workshop. Takes about 15–20 minutes.

0 Prerequisites 1 Create Folder 2 Init Git 3 CLAUDE.md 4 Co-work 5 Connect MCPs

Why do this before the workshop?

The workshop is hands-on from minute one. If your environment isn't set up, you'll spend the first hour watching instead of building. This guide walks you through everything step by step — no technical experience needed.

  • Follow the steps in order — each one builds on the last
  • If you get stuck, note where and we'll sort it at the start of the workshop
⏱ Estimated time: 15–20 minutes

⚙️ Before you start — install Claude Code

💻

Download Claude Code Desktop App

Go to claude.ai/download and download the app for your operating system (Windows or Mac). Install it like any other app.

🖼️
Screenshot: Claude Code download page
claude.ai/download — select your OS and click Download
🔑

Sign in with your Anthropic / Claude account

Open the app and sign in. Use the same account you use at claude.ai. If you don't have one, create a free account first.

🖼️
Screenshot: Claude Code sign-in screen
Click "Sign in" and use your claude.ai credentials
1

Create your Claude folder

A dedicated folder keeps all your Claude files, skills, and configs in one place

A
Create a new folder on your Desktop

Windows: Right-click anywhere on your Desktop → New → Folder. Name it my-agent.

Mac: Right-click on your Desktop → New Folder. Name it my-agent.

🖼️
Screenshot: Creating a new folder on the Desktop
Right-click → New Folder → name it "my-agent"
B
Open that folder in Claude Code

In Claude Code, click the folder icon in the top-left corner (or go to File → Open Folder). Navigate to your Desktop and select the my-agent folder.

💡

This tells Claude Code "this is your workspace." Everything you build lives here, and Claude will always know where to look.

🖼️
Screenshot: Claude Code — Open Folder dialog
File → Open Folder → select "my-agent" on your Desktop
2

Initialize Git

Git is like a "save history" for your folder — it tracks every change so you can always go back

ℹ️

Not technical? Don't worry — you only need to do this once, and you can just copy-paste the command below. Think of it as turning on autosave.

A
Open the terminal inside Claude Code

In Claude Code, look for a Terminal tab at the bottom of the screen (or go to View → Terminal). Click it to open a command window.

🖼️
Screenshot: Opening the Terminal in Claude Code
View → Terminal — a panel opens at the bottom of the screen
B
Run the git init command

Click in the terminal area, then type exactly this and press Enter:

git init

You should see a message like: "Initialized empty Git repository in..." — that means it worked.

🖼️
Screenshot: Terminal showing "git init" success message
You should see "Initialized empty Git repository in C:/Users/.../my-agent/.git/"
⚠️

If you see "git is not recognized" — Git might not be installed. Download it from git-scm.com and try again, or flag it for the workshop.

3

Create your CLAUDE.md file

This is Claude's memory — it tells Claude who you are, what you do, and how you like to work

ℹ️

CLAUDE.md is a text file that Claude reads at the start of every session. It's the difference between Claude acting like a blank slate every time vs. knowing your name, role, and preferences.

A
Use an AI chatbot to generate your CLAUDE.md

Open claude.ai or Gemini in your browser and paste in this prompt. Answer its questions naturally — the more detail, the better.

Copy this prompt

"I am setting up a personal AI agent. Ask me a series of questions about who I am, my job, my responsibilities, and how I prefer to communicate. Then use my answers to generate a CLAUDE.md file — a configuration file that will tell the AI agent everything it needs to know about me to be genuinely useful."

Once it generates the file, copy the full output.

🖼️
Screenshot: Claude.ai generating CLAUDE.md content
The chatbot will ask you questions, then produce a ready-to-use CLAUDE.md file
B
Save it as CLAUDE.md in your folder

In Claude Code, go to File → New File. Paste the content from the chatbot. Then save it as CLAUDE.md (all caps, exactly) inside your my-agent folder.

🖼️
Screenshot: Saving CLAUDE.md in Claude Code
File → New File → paste content → Save As "CLAUDE.md" in the my-agent folder
💡

The file name must be exactly CLAUDE.md — capital letters, no spaces. That's how Claude Code knows to auto-load it.

4

Enable Co-work & restart

Co-work unlocks collaborative features — it needs to be on before you start building

A
Enable Co-work in settings

In Claude Code, open Settings (gear icon or Claude Code → Settings in the menu bar). Look for Co-work or Cowork and toggle it on.

🖼️
Screenshot: Settings panel with Co-work toggle
Settings → find "Co-work" → toggle on
B
Restart Claude Code

Fully close and reopen Claude Code. This makes sure Co-work is active before your first session.

⚠️

Don't just close the window — make sure the app is fully quit (right-click the taskbar icon → Quit, or Cmd+Q on Mac).

🖼️
Screenshot: Fully quitting Claude Code
Right-click the Claude Code icon in taskbar/dock → Quit
5

Connect your apps (MCPs)

MCPs are connectors that let Claude read and interact with your real tools — Gmail, Drive, Calendar, Slack

ℹ️

What is an MCP? Think of it as a plug adapter. Claude Code can't access Gmail by default — but when you install the Gmail MCP, you give Claude permission to search and read your emails on your behalf. You stay in control: Claude asks before doing anything.

Connect these four — the first three are required, Slack is optional but recommended:

Google Drive
Required

Find and read documents, meeting notes, and decks from your Drive

Gmail
Required

Search and read emails; draft replies without leaving Claude

Google Calendar
Required

Read your schedule; use it in meeting prep and briefing skills

Slack
Optional

Read channels and DMs; surface context for meeting prep

A
Open MCP settings in Claude Code

In Claude Code, click the plug/connector icon in the left sidebar, or go to Settings → MCP Servers. This is where you manage all your app connections.

🖼️
Screenshot: MCP Servers panel in Claude Code
Settings → MCP Servers — you'll see a list of connected apps (empty to start)
B
Add Google Drive MCP

Click Add MCP Server (or the + button). Search for Google Drive and select it. Click Connect and sign in with your Google account when prompted. Grant the requested permissions.

🖼️
Screenshot: Adding Google Drive MCP
Search "Google Drive" → Connect → sign in with Google → Allow
C
Add Gmail MCP

Repeat the same process: click Add MCP Server, search Gmail, connect, and sign in. Use the same Google account.

🖼️
Screenshot: Adding Gmail MCP
Add MCP Server → search "Gmail" → Connect → sign in
⚠️

You'll see a Google permissions screen. Read it and click Allow — Claude needs this to be able to search your emails during the workshop.

D
Add Google Calendar MCP

Same again: Add MCP Server → Google Calendar → Connect → Allow. Same Google account.

🖼️
Screenshot: Adding Google Calendar MCP
Add MCP Server → search "Google Calendar" → Connect → Allow
E
Add Slack MCP (optional but recommended)

Click Add MCP Server → Slack → Connect. You'll be asked to sign in to your Slack workspace. Choose GetFabricTalk.slack.com.

🖼️
Screenshot: Adding Slack MCP
Add MCP Server → Slack → Connect → sign in to GetFabricTalk workspace
F
Set permissions to "Ask before acting"

For each MCP you connected, click the gear/settings icon next to it and make sure the permission mode is set to "Ask" or "Ask before sending". This means Claude will always ask you before sending any email or posting any message.

🖼️
Screenshot: MCP permission settings
Click the gear next to each MCP → set to "Ask" mode
💡

This is important — it means Claude drafts and plans, but you always have the final say before anything gets sent or changed.

✅

You're all set for the workshop!

If anything didn't work, make a note of where you got stuck — we'll go through it together at the start of the session. Even partially set up is fine.
